문제 상황

  • F&B 회사임에도 각 브랜드별 메뉴 판매량, 사이드 판매량, 메뉴와 사이드의 관계, 시계열 데이터의 부정확성 문제 존재
  • 영수증 데이터 수집이 웹 엑셀 다운로드 형태로만 가능하여 한 달치 데이터 다운로드에도 긴 처리시간 소요
  • 더 긴 기간의 데이터 수집 시 런타임 에러 발생으로 대용량 데이터 처리 불가능
  • 개발부서 부재로 직접적인 기술적 해결책 필요

해결 과정

  • 타 솔루션 업체와 API 연동 방안 협의했으나 커스텀 구축비용 500만원 필요
  • HTTP 통신 구조 분석으로 개별 엔드포인트와 파라미터 기반 데이터 수집 구조 파악
  • 데이터 사이즈 제한으로 인한 청크 단위 처리 방식 구현
  • Python requests 라이브러리 활용한 자동화 솔루션 구축

데이터 표준화 문제 해결

  • 메뉴 구조 문제: 메인메뉴, 옵션메뉴, 사이드메뉴의 획일화된 구조 부재 → 모든 패턴의 경우의 수 정리하여 일관된 구조 구축
  • 메뉴 식별 문제: 메뉴 이름의 시퀀스 부재, 메뉴 코드 없음 → 텍스트 포함 및 유사 텍스트 기반 메뉴코드 생성 및 매핑
  • 메뉴 뎁스 문제: 메뉴들의 뎁스 구조 일관성 부족 → Pandas 활용한 뎁스 분리 및 상단 row 생성

기술 스택

Python Requests Pandas HTTP API 데이터 표준화 자동화 스크립트 청크 처리 텍스트 매칭

핵심 원칙

  • 운영팀의 업무 프로세스를 방해하지 않는 범위에서 문제 해결
  • 현재 업무 프로세스를 완전히 뒤집어서 흐름을 방해하지 않고 기존 방식과 느낌 유지
  • 데이터 전문가가 아닌 판매 성장 전문가들의 업무 연속성 보장

성과 및 결과

  • 각 브랜드별 메뉴 판매량, 메인메뉴와 사이드메뉴의 관련성 데이터를 오차범위 2% 내외로 정확히 분석 가능
  • 메뉴별 시계열 데이터와 트렌드를 한눈에 파악할 수 있는 시스템 구축
  • 3개 브랜드의 영수증 데이터 표준화 문제 완전 해결
  • 개발부서 없이도 기술적 문제 해결 가능한 자동화 시스템 구축
  • 500만원의 커스텀 구축비용 절약 및 내부 솔루션으로 완전 자립
