DB가 없다? 공감이 필요해

Database Management & Team Collaboration

구글시트와 엑셀 중심의 비효율적인 데이터 관리 환경을 MySQL부터 BigQuery까지 확장하여 체계적인 데이터 레이크를 구축했습니다. 가장 중요한 것은 2년 만에 공감하는 동료들을 만들어냈다는 것입니다.

문제 상황

  • 기존 데이터 분석 업무환경이 구글시트를 메인으로 하고 저장은 엑셀로 하는 고전적인 방식
  • 영수증 데이터를 비롯한 다양한 도메인의 데이터를 관리하기에는 비효율적이고 확장성 부족
  • 적어도 DB 스토리지 및 SQL로 관리가 시급했지만 공감할 수 있는 동료는 0명
  • 데이터 관리의 심각성과 해결의 필요성을 공론화할 방법이 필요

해결 과정

  • 데이터가 이렇게 처리될 수 있다는 것을 보여드리기 위해 개인 PC에서 MySQL 설치 및 운영
  • DB 전문가가 아닌 상황에서 실무에 필요한 내용을 학습하여 적용
  • 각 도메인별로 데이터 모델링과 정규화 작업 수행
  • 다른 팀과 공유하려면 DB 포트 오픈, 보안, 교육 등의 문제가 시급함을 인식

팀 내 도입 전략

  • 일단 우리 팀원들과만 사용해보기로 결정
  • 개발팀, IT팀이 없는 상황에서 아이피를 논리적으로 고정
  • SQL을 천천히 교육하고 차근차근 각자 필요한 데이터를 테이블로 만들어서 관리하게 함
  • 팀원들이 직접 테이블을 만들고 관리하는 과정을 통해 데이터베이스의 장점을 체감

시스템 확장

  • 팀원들이 만든 테이블들을 전체적으로 표준화하여 스키마 정리
  • 테이블 간의 관계를 통해 ERD 구축
  • 구글 생태계를 가진 회사이므로 GCP를 활용해서 BigQuery까지 확장
  • 구글시트에서 바로 불러올 수 있게 구축
  • 하루에 수집되는 데이터가 100만개 이상, 이제는 거대한 데이터 레이크가 생김

기술 스택

MySQL SQL Database Design Data Modeling ERD Schema Design Data Normalization Team Training GCP BigQuery Data Lake

최종 성과 - 공감을 불러일으키다

  • 가장 중요한 성과: 2년 만에 공감하는 동료들이 서서히 늘어남
  • 구글시트와 엑셀 중심의 비효율적인 데이터 관리 환경을 MySQL 기반으로 전환
  • 팀원들이 직접 SQL을 사용하여 데이터를 관리할 수 있는 역량 확보
  • 각 도메인별 체계적인 데이터 모델링 및 정규화 완료
  • 전체 테이블 표준화 및 ERD 구축을 통한 데이터 관계 명확화
  • GCP BigQuery를 활용한 확장 가능한 데이터 레이크 구축
  • 하루 100만개 이상의 대용량 데이터 처리 시스템 완성
  • 데이터 관리의 새로운 패러다임을 팀 내에 정착
