프로젝트 개요

KiyoungFnB의 복잡한 하이브리드 인프라 환경을 통합하는 데이터 플랫폼을 구축했습니다. GCP 클라우드 영역에서는 Jenkins CI/CD(Compute Engine 34.4), Apache Airflow, BigQuery를 구축하고, KiyoungFnB 내부 시스템에서는 MySQL, PostgreSQL 데이터베이스와 내부 Airflow를 운영합니다. 다층 서버 환경(Inbound Servers Master, MiniPC Slave, iwinv Servers Slave)과 모바일 앱 연동(Coupang, Baemin, NaverMap)을 통해 완전한 온프레미스-클라우드 하이브리드 데이터 아키텍처를 실현했습니다.

기술 스택

Google Cloud Platform Compute Engine (34.4) BigQuery Google Cloud Storage Jenkins Nginx Gunicorn Django Apache Airflow MySQL PostgreSQL Coupang API Baemin API NaverMap API Looker Google Sheets

주요 기능

  • 하이브리드 아키텍처: GCP 클라우드와 KiyoungFnB 온프레미스 환경 통합
  • 데이터 추출(Extract): API, Files, DB, Other Cloud에서 다중 소스 데이터 수집
  • 데이터 변환(Transform): Jenkins CI/CD와 Airflow를 통한 실시간 데이터 파이프라인
  • 데이터 로드(Load): BigQuery, MySQL, PostgreSQL 다층 스토리지
  • 모바일 데이터 수집: Coupang, Baemin, NaverMap 실시간 연동
  • 서버 오케스트레이션: Inbound Master → MiniPC/iwinv Slave 구조
  • Compute Engine 관리: Jenkins → Nginx → Gunicorn → Django 스택
  • BI 보고 시스템: BigQuery → Looker/Google Sheets/Web 분석
  • 내부 데이터 통합: KiyoungFnB MySQL/PostgreSQL 관리
  • Git 기반 배포: 코드 버전 관리와 자동 배포 파이프라인

성과 및 결과

  • 하이브리드 인프라 통합: 온프레미스-KiyoungFnB와 GCP 클라우드 완전 통합 달성
  • 다중 데이터 소스 통합: API, Files, DB, Other Cloud의 실시간 동기화
  • 모바일 앱 생태계 연동: Coupang, Baemin, NaverMap 완전 통합 데이터 수집
  • Compute Engine 최적화: Jenkins-Nginx-Gunicorn-Django 스택 성능 향상
  • 서버 클러스터 관리: Master-Slave 구조로 가용성 및 확장성 확보
  • 빅데이터 처리 혁신: BigQuery 기반 대용량 분석 및 실시간 뷰
  • 내부 시스템 혁신: MySQL-PostgreSQL 이중화 및 Airflow 오케스트레이션
  • BI 대시보드 구현: Looker, Google Sheets, Web 기반 실시간 의사결정 지원
